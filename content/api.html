get /export
Export full detail of transactions for one or more addresses. (accountsExport)
Export full detail of transactions for one or more addresses.

Query parameters

addrs (required)
Query Parameter — one or more addresses (0x...) to export
topics (optional)
Query Parameter — filter by one or more log topics (only for --logs option)
fourbytes (optional)
Query Parameter — filter by one or more fourbytes (only for transactions and trace options)
appearances (optional)
Query Parameter — export a list of appearances
receipts (optional)
Query Parameter — export receipts instead of transaction list
statements (optional)
Query Parameter — for use with --accounting option only, export only reconciliation statements
logs (optional)
Query Parameter — export logs instead of transaction list
traces (optional)
Query Parameter — export traces instead of transaction list
accounting (optional)
Query Parameter — export accounting records instead of transaction list
articulate (optional)
Query Parameter — articulate transactions, traces, logs, and outputs
cache_txs (optional)
Query Parameter — write transactions to the cache (see notes)
cache_traces (optional)
Query Parameter — write traces to the cache (see notes)
skip_ddos (optional)
Query Parameter — toggle skipping over 2016 dDos transactions ('on' by default)
max_traces (optional)
Query Parameter — if --skip_ddos is on, this many traces defines what a ddos transaction is (default = 250)
freshen (optional)
Query Parameter — freshen but do not print the exported data
factory (optional)
Query Parameter — scan for contract creations from the given address(es) and report address of those contracts
emitter (optional)
Query Parameter — for log export only, export only if one of the given export addresses emitted the event
source (optional)
Query Parameter — for log export only, export only one of these addresses emitted the event
relevant (optional)
Query Parameter — for log and accounting export only, if true export only logs relevant to one of the given export addresses
count (optional)
Query Parameter — only available for --appearances mode, if present return only the number of records
first_block (optional)
Query Parameter — first block to process (inclusive)
last_block (optional)
Query Parameter — last block to process (inclusive)
first_record (optional)
Query Parameter — the first record to process
max_records (optional)
Query Parameter — the maximum number of records to process before reporting
clean (optional)
Query Parameter — clean (i.e. remove duplicate appearances) from all existing monitors
staging (optional)
Query Parameter — enable search of staging (not yet finalized) folder
unripe (optional)
Query Parameter — enable search of unripe (neither staged nor finalized) folder (assumes --staging)
load (optional)
Query Parameter — a comma separated list of dynamic traversers to load
reversed (optional)
Query Parameter — produce results in reverse chronological order
by_date (optional)
Query Parameter — produce results sorted by date (default is to report by address)
summarize_by (optional)
Query Parameter — for --accounting only, summarize reconciliations by this time period
